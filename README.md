# Vape Shop Telegram Mini App 💨

Современный интернет-магазин вейпов внутри Telegram. Бэкенд на **FastAPI**, бот на **aiogram 3.x**, база данных **MySQL (SQLAlchemy 2.0)**.

---

## 🚀 Подготовка и Запуск

1. Установите зависимости: `pip install -r requirements.txt`
2. Настройте файл `.env` (скопируйте из `.env.example`).
3. Наполните базу данных тестовыми данными: `python scripts/seed_db.py`
4. Запустите проект: `python main.py`

---

## 💻 Локальное тестирование Mini App

Во время разработки вы можете тестировать Mini App двумя способами:

### Способ А: Тестирование прямо в Telegram (через Ngrok)
Это самый надежный способ проверить реальную работу приложения на телефоне:
1. Запустите ngrok для проброса порта: `ngrok http 8000`
2. Скопируйте полученный HTTPS URL (например, `https://a1b2-c3d4.ngrok-free.app/web/index.html`).
3. В файле `.env` укажите этот URL в переменную `MINI_APP_URL`.
4. Запустите проект (`python main.py`), откройте своего бота в Telegram, введите `/start` и нажмите "Открыть магазин".

### Способ Б: Тестирование в браузере (Отладка верстки)
Полезно, если вам нужно инспектировать элементы (F12) или проверять API без Telegram:
1. В файле `api/dependencies.py` временно **закомментируйте** логику проверки криптографии Telegram и просто верните моковый словарь:
```python
def verify_telegram_webapp_data(...) -> dict:
    # ВРЕМЕННЫЙ МОК ДЛЯ ТЕСТИРОВАНИЯ В БРАУЗЕРЕ
    return {"id": 123456789, "username": "test_user"}
```
2. Запустите `python main.py`.
3. Откройте в браузере: `http://localhost:8000/web/index.html`

> ⚠️ **ВНИМАНИЕ:** Обязательно верните логику верификации `verify_telegram_webapp_data` обратно перед деплоем на рабочий сервер!
